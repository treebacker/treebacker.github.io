为什么记录

在学习pwn的入门过程中，这一系列的练习给了我很多的帮助，包括踩坑。让我更加直观地体验到了构造ROP的乐趣。学习到了基本的pwn思维。本篇文章不作为学习的参考，只是记录我的轨迹。

0x01 ret2win

**Idea:**

这里直接给出了一个cat flag的函数。我们只需要用这个函数的地址覆盖pwnme函数返回地址即可。

ret2win32

ret2win64

0x02 split

**Idea:**

程序存在栈溢出，并且提供了system函数，通过查找也能找到cat flag.txt的指令。所以很明显的构造payload即可。

Split 32bit

Split 64bit

0x03 callme

Idea:

程序存在栈溢出，并且在usefulFunction中调用了三个函数，callmeone\two\three。可以发现这三个函数来自.so文件。查看文件得知，三个函数都需要三个参数1、2、3.分别完成打开加密文件、打开key1文件、打开key2文件并输出flag的功能。所以我们需要尝试控制返回函数的参数（注意形参在栈中的位置）。

callme 32bit

callme 64bit

0x04 Wite4

**Idea:**

这次，并没有给我们有用的string。但是我们有system。所以，目的很明确，想办法写入自己的string。

除了read、get、等我们见到的读入函数外，在rop里面常用到的指令级别的读入，就是pop|mov|ret的组合了。

而写入的地址，最常见的就是.bss段。除此，我们也可以vmap查看一下内存的各部分权限（读写）自己选择合适的写入位置（这里仍然是.bss段）

Write4 32bit

Write4 64bit

0x05 badchars

**Idea:**

有时候，程序会对输入的payload进行检查，诸如"/bin/sh"的字符串很可能被检测到而进行破坏。

这时候，我们需要把payload进行“加密”，执行getshell操作前，再进行“解密”。最常用的加密操作就是xor了，并且加解密同用一段ROP链。

badchars 32bit

badchars 64bit

0x06 fluff

**Idea:**

有时候，仔细地查看汇编代码，运用一些工具。会发现一些很有趣的rop。

比如 xx xor 0 == xx。

fluff 32bit

fluff 64bit

0x07 pivot

**Idea:**

这次，fgets只允许我们读进有限的字符（没法布置足够长的payload在栈区执行），但是提供了另一个缓冲区的地址，这个缓冲区位于堆（heap）。通过寻找有用的ROP，我们可以巧妙地将payload布置在heap上，执行。

总结

1、32和64的程序，区别开来。传参方式，甚至还有x64的地址很可能存在\x00截断。

2、环境。之前用的ubuntu18.04，自己所有的x64的exp都不起作用，存在”在调用c运行库时找不到文件的问题”，索性换了ubuntu14.04。

3、GDB调试利器，加上peda真的好用。
