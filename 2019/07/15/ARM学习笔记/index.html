ARM学习笔记

    • 配置公钥，无密码ssh登录
    • 更新源

    • pwntools

数据类型

  • 不同于i386，ARM只支持四种数据类型（bit, byte, half word, word）
  • 数据类型可以以不同的形式出现（hex, binary, decimal, ascii)

  • 在指令中出现的立即数，前面要加上’#’b

    在指令中出现的立即数，前面要加上’#’标识；特别的在2-9进制的表示中加’n_’表示n进制数；如#2_01010
  • ARM所有的寄存器都是32bit，即在ARM里，word就是32bit（在i386是16bit)
  • 同样每种数据类型分无符号和有符号

寄存器

  • ARM的特点就是寄存器多（而相应的指令集少）
  • 通用寄存器R0-R12，简称GPSR
  • R13是stack pointer (x86下的 ebp)
  • R14是Link pointer（当调用子过程时，保存返回地址）
  • R15是Program pointer （pc）
  • CPSR是标志寄存器（后四者又称 SFRs)

指令集

  • 指令格式
    • instruction destination, source1, [source2]
    • 其中source1必须是寄存器；source2可以是寄存器或者立即数
    • 在某些指令下，source2的立即数取值只能在8bit内（例如mov指令）
  • Registers and ALU

  • 常见指令规则

    • 特别的，mutiple

      ldm、stm

      • 格式一般如下
      • ldm register_source, {var1, var2, ……}
      • stm register_destination, {var1, var2}
      • 其中，ldm、stm可以规定地址的变化方向，默认ia
        • ia（increase after)
        • ib (increase before)
        • da (decrease after)
        • db (decrease before)
      • 编译器会把ia优化省略

    • push pop指令

      • push {var1, var2, ……} （从右至左依次入栈）

      • pop {r1, r2, ……} (从左至右依次出栈)

      • ldmia sp! <==> pop
      • stmdb sp! <==> push

  • 指令长度
    • 在ARM模式下，都是4byte长度
    • 在thumb模式下，可以是2byte， 也可以是4byte

  • 伪指令

    • 让链接器识别处理的

    • AREA

      • 定义一个段，可以是data, 也可以是code； 同时可以注明段属性（RWX）

      • 规则如下

        • AREA sectionname, attribute, attribute, …

        • 常见的属性有CODE\DATA\READWRITE\READONLY\COMMEN\ALIGN

    • END
      • 标记文件的结束
    • ENTRY
      • 注明程序的入口
    • EQU
      • 为寄存器、常量值定义符号名称
    • INCLUDE
      • 链接包含的文件
    • LDR R7, =#0xFFFFFFFF (用于赋值大于0xff的情况)

    • 实例

内存映射

  • pc寄存器寻址空间4G(32bit)

  • 4G内存空间可分为5个区域

    • 片上外设和IO寄存器； 也即ARM使用内存映射I/O

    • SRAM ; 存放stack和变量data数据 (断电消失)

      过大时，其地址管理有困难（编写arm汇编时），因此需要c编译器来处理

    • EEPROM； 存放代码（code）和关键系统数据（critical system data)（片上可以没有这个区）

    • FLASH空间； 存放代码数据（ROM code）、静态常量字符串、查询表

    • DRAM； 实际是RAM的扩展，被设计为既可以存放data, 也可以存放code（同x86)

      • 样例

  • load and store
    • LDR Rd, [Rx]; 把Rx地址处的4byte内容 载入到 Rd寄存器
    • STR Rx, [Rd]; 把Rx寄存器的值 存储进 Rd地址处
    • LDRB同LDR，STRB同STR； 但是只copy一个byte，并将Rd的高3byte清零
    • 类似的，LDRH\STRH
  • 程序状态寄存器

    • 除了与x86类似的N\C\Z\V等外

    • 有T标志Thumb模式

    • 在指令的后面加S（如ADDS、SUBS）即可将指令执行的影响在CPSR上有所反应。（改变CPSR）

    • 注意与X86的不同，并不是所有的执行都影响CPSR，且可由我们选择。

    • 常见的指令对CPSR的影响

    • CPSR的标志位用于条件跳转

    • 一般的条件跳转

    • Thumb模式下的条件跳转

      • 最多支持4层
      • if-then-else模式
      • then下相同数量的条件指令的逻辑与then的条件一致；相同数量的条件指令的逻辑与其相反

    • 直接跳转branch

      • B无条件跳转
      • BL跳转，并将pc+4存入LR寄存器（相当于x86下的call指令）
      • BX和BLX同BX和BL，另外切换ARM和Thumb模式

函数与栈

  • 栈的四种状态

  • 对应的不同的mutl指令

  • 参数传递规则

    • 前四个参数一次存入r0, r1, r2, r3； 超过部分入栈传递

  • 建立栈帧 r11（类似于x86下的ebp）

  • 返回值存入r0 (类似于x86下的eax)

  • 函数系统调用号存于r7（类似于x86d的eax）

  • 不同于x86的int 80，arm使用 svc #0或者svc #1

开发shellcode

  • 开发shellcode的四个基本流程

    • 知道我们需要用的函数的系统调用（比如system在OS层调用execel）；这里如果不清楚可以strace跟踪看到。

    • 得到相应的系统调用号；该信息保存在/usr/include/arm-linux-gnueanif/arm/uninstd.h下

    • 一般写出来的shellcode会有null字节，我们需要去除（在arm里最好使用thumb模式开发shellcode）

      • 当然系统调用就使用svc #1而不是svc #0了；

      • 除了模式，还可以有指令等效替代；

      • 字符串的null结尾字符，可以使用strb指令修改。（当然该方式需要.code可写)

    • 转化shellcode为hex字符串

      • objcopy -O binary exefile exefile.bin

  • 注意在Thumb模式下，指令长度对齐4，否则会出错（adr指令）；

    • 对齐可由mop指令填充实现

  • bind shell和reverse shell

    • 前者主机向目标发起连接，很大几率被防火墙阻拦
    • 后者目标机向主机反弹连接，主机可以控制受害机
