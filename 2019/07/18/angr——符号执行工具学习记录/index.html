angr ——符号执行工具学习记录

概念

  • 二进制文件生成Project
    • Project有一些基本属性：CPU架构，文件名，入口地址
  • 加载器
    • 工程文件project的属性loader来访问
    • 可以查看程序以及程序加载的共享库、以及加载的地址空间
  • The factory

    • project的一个类，其中有经常会使用的构造函数

    • Blocks

      • project.factory.block()提取代码块
      • 可以访问代码块的一些基本信息
      • 反汇编、地址、指令
    • States 符号执行时，程序的状态的概念很重要。
      • Project只是程序的一个初始镜像
      • 通过特殊对象SimState可以在分析程序时实时获得程序的状态。
      • project.factory.entry_state()
      • state保存了程序的寄存器、内存、文件系统数据……

    • 符号执行中的数据

      • 不同于python int。angr使用bitvector保存数据
      • bitvector有两个属性：数值、数值的bit长度(.length属性)
      • 两者可以相互转化BVV(value, length)和eval(bv)
      • 不同长度的bitvector不可以正常的运算，但是可以扩展到一样的长度
    • 模拟管理器（Simulation Managers）
      • 为了尝试多种路径，能够保存原来的状态，使用manager，可以在manager中模拟执行；而不会影响state
      • 可以针对某一个状态单独前进.step()，也可以查看分支的数目len(.active)，也可以执行到结束.run()
      • 可以将多个状态组在一起成一个stash，每个stash都是一个state列表
      • 几个重要的stash的类型
        • active 当前stash中仍然有可以继续执行的state分支
        • deadended 保存所有执行结束的state的分支
        • pruned
        • unconstrained 不受约束的（用户输入、、、）
        • unsat 不可能满足的状态集合
      • errored状态，record.debug()可以调试查看
    • Analyses 分析
      • 内置分析，快速分析，多种分析
      • 提供的分析结果可供参考

加载二进制文件

  • 加载器

    • 将整个二进制文件以及加载的共享库导入虚拟地址空间

      • main_object 程序主体
      • shared_objects 共享链接的文件
      • extern_object angr内部地址、和未解析的导入
      • kernel_object angr系统调用的提供地址
      • find_object_containing(address) 获得给定地址的对象的引用
    • 可以获得程序的很多信息
      • 地址块、范围
      • ELF和PE文件的对象
      • segment，sections的信息
      • 获得对给定地址的对象的引用
      • 获取PLT\GOT表项
      • 获取链接的基础以及实际映射的内存位置
  • 符号与重定位
    • 符号是在可执行文件中的一种将名称与地址映射起来的机制
    • 符号最重要的三个属性就是名称、所有者和地址。其中地址有三种方式
      • .rebased_addr 全局空间地址
      • .linked_addr 相对二进制预链接基地址。如reaelf的输出
      • .relative_addr 相对于对象的基地址，RVA
    • symbol分为在ELF主文件main_object中和在共享文件中；同一个symbol，在前者是import；在后者是export
  • 程序的状态state
    • 多种状态
      • .blank_state() 块状态。构造一个空白块，数据未初始化
      • .entry_state() main的入口状态
      • .full_init_state() 在main之前的状态，完成动态库的加载等操作
      • .call_state() 构造一个准备执行一个函数的状态
    • 状态的构造
      • 每种构造函数均可以传一个.addr参数作为起始地址
      • 需要的时候，可以为.entry_state()和full_init_state()传入参数列表args和参数个数argc；但是需要注意argc不大于args的个数
      • .call_state(addr, arg1, arg2, arg3, ……)，指针需要angr.PointerWrapper()包装
  • 基本的执行

    • state.step()

      • 返回值是一个successor state的列表

    • 我们将对模拟程序的执行的stdin重定向到angr提供的SimFile。

      • 可以使用state.posix.stdin.load(0, state.posix.stdin.size)来检索内容。

    • .explore()带上一个fand()参数，即可以使得程序一直运行直到达find参数列表下的地址（函数）；满足的状态加入.found的stash中

      • find的参数可以是地址、函数、或者avoid的地址、函数
      • .posix.dumps(0), dumps(1), dumps(2) 分别指向程序的stdin, stdout, stderr；
      • 可以用以上操作来判断输出中是否包含敏感字符串（明显地标志着到达某一状态），并取得此时的stdin输入

    • 调用.step()实际上，在angr实现上，会按照一定的规则，依次遍历可用的引擎来处理状态运行，将结果保存在SimSuccessors中

      • successors的类型
        • successors 满足条件的状态
        • unsat_successors 不满足约束的状态（永远不会成立）
        • flat_successors 满足条件的状态且经过处理（扁平化，确定化，具体化：可能的值的数目在256下）
        • unconstrained_successors 满足条件的状态，但是不受约束，或者可能的值的数目在256以上
        • all_successors 以上1, 2, 4种状态的合集

    • 探索技术（遍历分支状态的方式）

      • DFS深度优先（默认即是）
      • Explorer，函数.explore()，允许你搜寻目的地址和避免某些地址
      • LengthLimiter 设置状态路径的最大长度
      • LoopSeer 设置最大循环数目，超过该次数的状态将被暂时放弃，等到其他状态搜寻完毕再次继续这些状态
      • ManualMergepoint 标记某一地址，到达该地址的状态会整合到一起
      • ……

    • 断点Breakpoints

      • 可以在程序的很多状态前、后下断点，且定义自己的callback函数

常用操作

  • explore
    • simgr.explore(find, avoid)
      • 当正确路径与错误路径分支明显（都集中在一个或几个），可以直接罗列地址，find与avoid分别为正确分支地址与错误分支地址
      • 反之，如果分支相对混淆交叉，则可利用stdout输出的差异，利用回调函数进行判断，即find_condition & avoid_condition
  • Surveyors
  • Path Group
  • Symbolic args
  • Symbolic input
  • Breakpoint
  • Hook
