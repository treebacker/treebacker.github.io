利用IO file进行pwn

简述

  • glibc对所有用户产生的FILE结构进行维护，内存分配在heap中，且以链表形式维护（FILE有chain域），io_list_all为链表头。

  • 但是在FILE结构外另有io_file_plus结构

  • 其中vtables保存着标准io函数调用的函数指针

  • 我们能够利用的方式就是三个思路，要么覆盖真实的vtables里的函数指针，要么伪造vtables；再或者针对链表io_list_all，该值时libc.so的全局变量，如果我们伪造一个，就可以控制伪造file结构及vtables，这就是FSOP攻击（当然需要一些验证）

例题

    • 漏洞分析，存在一个任意地址写漏洞，可以5次，每次1byte。

      • 利用IO FILE，在exit之后，会调用file_list_all里的函数setbuf。如果我们可以伪造setbuf为one_gadgets就可以利用。

  • 利用思路B
