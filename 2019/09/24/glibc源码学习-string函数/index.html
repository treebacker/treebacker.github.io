glibc源码学习-string操作函数分析

strlen函数

  • 返回null结束的字符串的长度。

  • 但是有着非常有意思的实现方式，不同于我们的逐个字节的比较寻找null字节。该实现可以每次比较一个4bytes（32bits），8bytes（64bits）。

  • 主要是依赖于两个魔数.(64bits时需左移扩展)

  • 这两个模数与任意一个4bytes数子tmp运算有如下规律

  • 其原理是借助于借位和符号位. 0可以保留进位的空间,1可以传递进位的效果.

  • strcpy函数

    • 内部通过memcpy实现的，可以看到nbytes是src得长度+1，实际存在一个null字节的覆写。

  • memcpy函数

    • 内部实现分为三种复制方式, 逐字节复制和PAGE复制以及WORD复制。用以尽可能地提高效率。

      • 为实现src地址和dst地址WORD对齐，可能需要进行逐字节复制。

      • 对齐后，尝试进行PAGE复制。因为当所需复制得数目超过阈值0x400时，借助操作系统得虚拟内存空间复制效率更高。（当然会检查地址是否页对齐）

  • 值得注意的是,并没有检查dst地址的空间是否充足,换句话说,memcpy和strcpy函数均存在缓冲区溢出的潜在危险.

  • strcat函数

    • 内部实现是通过strcpy函数

  • strcat函数覆盖原dst的null字节，同时也会把src的null字节复制过去。

  • strcmp函数

    • 由低地址向高地址逐字节比较两个字符串。

  • strncmp函数

    • 该函数遇到’\x00’停止检查，在很多比较场合可以用来绕过检查。
