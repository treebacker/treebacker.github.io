Linux命令学习笔记

第一天——Cat

  • cat读取文件（一个或者多个），将文件内容拼接在一起并将结果发送到stdout
    • cat file1 file2 file3……
  • 管道操作符结合
    • output_from_commands | cat
    • echo “treebacker” | cat - file.txt
      • 将stdin和文件内容拼接输出
      • 这里的 - 作为stdin的文本文件名
  • 去掉多余的空白行
    • cat -s file
  • 输出行号
    • cat -n file
  • 不可以用重定向来覆盖输入文件，下所示是错误的
    • cat -n filename > filename
  • 使用管道重定向到输入文件会清空输入文件
    • cat -n filename | cat > filename
  • 同样的 > 覆盖文件， >>续写文件也适用于cat
    • cat > file （如果不存在file，就新生成文件file）
    • cat >> file

第二天——Find

  • 从指定位置找所有文件以及子目录

    • find base_path -print
      • base_path可以是任意位置，find从该位置往下查找
      • -print 选项以\n分割输出

  • 基于文件名的通配符或正则表达式进行搜索

    • find base_path -name ‘*.txt’ -print
      • -iname 忽略大小写
      • 支持逻辑操作 -a -o（and 和or）
    • find base_path -regex ‘.*(.py|.sh)$’
      • -regex即正则表达式
      • -iregex可以在匹配时忽略大小写

  • 否定参数（在匹配调价前加上”!”）

    • find base_path ! -name “*.txt” -print

  • 基于文件时间戳进行搜索

    • 三种时间戳

      • 访问时间(-atime)：用户最近一次访问时间（access)
      • 修改时间(-mtime)：用户最后一次修改时间（modify）
      • 变化时间(-ctime)：文件元数据（权限、所有者）最后一次修改时间（change）
      • 以上三种参数前加修饰’-‘或’+’，分别表示小于或大于

    • 打印最近7天内被访问的所有文件

    • 打印恰好在7天前访问的文件

    • 打印出访问时间超过7天的所有文件

    • 时间除以“天数”论外，也可以以分钟计算同上（-amin, -mmin, -cmin）

  • 基于文件大小搜索（-size)

    • 大小大于2k的文件

    • 可用的大小单位b，c， w，k，M，G

  • 基于文件的权限查找（-perm）

    • 打印出所有744权限的文件

  • 基于文件的所有权进行查找（-user USER)

    • 打印出所有root用户的文件

  • 其他

    • 删除匹配文件（-delete）

      • 删除当前目录下的.py文件

    • j结合其他命令执行（-exec）

      • 将匹配到的root的文件改给tree

第三天 —— xargs

xargs命令重新格式化stdin接受到的数据，应该紧接在管道操作符之后。xargs默认执行echo命令

  • 一般的输出格式化

  • 默认空白字符为分隔符，-d参数指定分隔符

第四天 ——qemu

qemu是一款模拟处理器自由软件；经常用来模拟arm、mips等各种架构的操作环境。

记录一些经常用到的

  • 用户模式下执行程序

    • qemu-arm、qemu-mips、qemu-mipsel等等

    • 在某一环境下，运行程序命令格式“qemu-arm [options] program [arguments]

      • options有：-E 设置环境变量；-g 设置gdb连接的端口；LD_PRELOAD=newlib 使用指定的newlib劫持系统调用

  • 系统模式

    • 根据内核、磁盘重新生成一个系统

      • 命令格式 ”qemu-system- mispel [options] disk_image”

      • 网络配置

第五天——chroot

  • 作用

    • 生成一个独立的文件系统，（自定义根目录）
    • 我们需要将该系统内部对原系统的依赖copy进来

  • 场景举例

    • 在研究qemu模拟系统的时候，arm架构或者mips架构的程序运行
    • 在获得固件的文件系统之后，由于原系统的内部程序对库的依赖，全是根目录下；若想正常模拟，必须将文件系统目录指定为根目录

  • 举例

    • 将boot-fire作为根目录，qemu模拟运行boot-fire/bin/boa程序

第六天——tr

  • 将一组字符转换为另一组字符（translate）

    • 只能通过stdin接受命令行参数；set1与set2字符依次对应；set2多余不计，少则最后一位补上

    • set在连续时，可简写“起始字符-终止字符”

  • 实战演习

    • 输入中的字符大写转为小写

    • 数字简单加、解密

    • 一句话的ROT13加密(加解密同)

  • 其他用法

    • tr删除字符 tr -d [set]

    • 字符组补集 tr -c set1 set2

      • set1必有，set2可选

      • -c时，两者都得有

      • -d时，只能由set1

    • tr压缩字符（删除重复出现的字符） tr -s [待压缩的字符集合]

第七天——校验和与核实

  • MD5

    • 用法 md5sum filename …

    • 保存文件的md5备份；定时比较是否变化

  • SHA-1
    • 用法同md5sum，这是sha1sum

第八天 ————scp

  • 基于ssh与cat命令和管道的实现

  • 在主机之间传送数据

  • 参数格式

    • scp 参数 原路径 目标路径

    • 常用

第九天——sort

  • sort命令可以对文本文件和stdin进行排序；uniq通常配合使用，提取不重复的行。

    • 排序一组文件

    • 按照数字排序

    • 按照逆序排序

    • 按照日期排序（小在前）

    • 合并两个已经排序的问价

    • 找出已经排序的文件中不重复的行

    • 检查文件是否已经排序过

    • 指定列
